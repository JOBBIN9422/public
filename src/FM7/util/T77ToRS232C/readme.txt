Delphis (COMPAC)
実機FM77AV40で確認。ゲーム開始、キーで自機を操作するところまで確認。
デフォルトインストールアドレス、ブリッジアドレスのままで動作。

ただし、次のF-BASICプログラムで起動する必要あり。

10 CLEAR ,&H14FF
20 LOADM"DATA"
30 POKE &H68B1,&HFD
40 POKE &H68B2,&H05
50 POKE &H68B3,&H2A
60 POKE &H68B4,&HFB
70 EXEC
80 LOADM"DELPH.M"
90 POKE &H560D,&HFD
100 POKE &H560E,&H05
110 POKE &H560F,&H2A
120 POKE &H5610,&HFB
130 EXEC

最初AV40で単純に RUN "" で実行しようとしたら3〜5回に1回ぐらいしか起動に成功
しないので逆アセンブルを見たらサブCPUがかなりシビアなタイミングでHALT要求に
応答することを前提に書いてあった。

LDA #$80, STA$FD05して次のクロックでサブCPUがHALTしてるなどということがあり
得るのだろうか？当時よくこれで動いてたなあ。

本当は少なくとももう一か所(サブCPUにプログラムを転送するところ)あるのだが、
直せない。が、どうやらこの二か所を直しておけば、5回連続起動できたから大丈夫
のようだ。

多分、素のFM-7だとそのまま RUN "" で実行できるんだと思う。FM-7も実機がある
から試したいところだけど、権力が無いため大学に広いオフィスをもらうなんてい
うことができないため引っ張り出さないと実験できないので、次に引っ張り出した
ときに忘れなかったら確認する予定。
